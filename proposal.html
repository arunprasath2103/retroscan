The Problem
01 — Problem Statement

Why Retroreflectivity
Monitoring Is Broken Today

India's 1,46,000 km national highway network has thousands of signs and lane markings. Current measurement methods are dangerously inadequate.

⚠️

Handheld Devices Are Dangerous

Technicians must stand on live high-speed expressways to take readings — creating serious safety hazards for both workers and passing motorists.

🐌

Impossibly Slow Coverage

Manual measurement covers only a tiny fraction of the network per day. Most signs go years without any measurement — failures accumulate invisibly.

💸

Equipment Costs ₹15–50 Lakh

Commercial retroreflectometers are prohibitively expensive. NHAI can afford very few units, creating massive coverage gaps across the network.

🌧️

No Night or Wet Condition Data

Most critical failures occur in wet or night conditions — precisely when current handheld tools are least usable and readings least reliable.

The Solution
02 — System Design

How RetroScan AI
Works

Five tightly integrated steps transform a moving vehicle into a continuous, intelligent retroreflectivity monitoring station.

1

Controlled Pulsed Illumination

A 50W COB LED array fires precise bursts of white light at the road surface. The camera captures one frame with LED on, one with LED off, and subtracts them — completely eliminating ambient light interference. This is the same principle used in ₹50 lakh laboratory-grade devices.

50W COB LED Pulse Driver Circuit Ambient Subtraction
2

Calibrated Camera Capture

A 12MP machine vision camera is mounted at the IRC-standard observation angle (0.2° from illumination axis). The geometry is mechanically fixed so every reading is geometrically comparable — no calibration drift between measurements.

IMX477 12MP IRC 67/35 Geometry Fixed Optical Mount
3

AI Regression Model Predicts RL Value

A CNN regression model trained on road markings with known mcd/m²/lux values takes image luminance + ambient light level + vehicle speed as inputs. It outputs the estimated retroreflectivity value with separate condition branches for dry/wet/night/foggy environments.

CNN Regression TensorFlow Lite Edge Inference Multi-condition Model
4

Standards-Aware Decision Engine

Every predicted value is instantly compared against IRC 67 (road signs) and IRC 35 (pavement markings) thresholds. The system classifies each marking: Pass (green ≥ threshold), Warning (yellow within 20% of threshold), or Fail (red below threshold). Fail events trigger immediate alerts.

IRC 67 Thresholds IRC 35 Thresholds 3-tier Classification
5

GPS Tagging, Logging & Cloud Dashboard

Every reading is GPS-tagged in real time using a u-blox NEO-M8N module (accuracy ±2.5m). Data streams to a cloud dashboard that NHAI maintenance teams access to plan repairs. The live in-vehicle display shows a colour-coded heat map of the road ahead.

u-blox NEO-M8N Cloud Sync Live Heat Map Maintenance Alerts
AI / ML Core
03 — Artificial Intelligence & Machine Learning

The AI Engine Inside

The ML core is what transforms a simple camera into a calibrated measurement instrument — making the system adaptable, improvable, and scalable.

🧠 Model Architecture

  • Base: MobileNetV2 backbone (optimised for edge devices — runs at 15 fps on Jetson Nano)
  • Head: Custom regression layers output continuous mcd/m²/lux value instead of class labels
  • Inputs: Subtracted image frame (640×480) + ambient lux reading + vehicle speed + condition flag (dry/wet/night)
  • Output: Predicted RL value + confidence interval
  • Quantisation: INT8 quantised TFLite model for 4× faster edge inference

📊 Training Strategy

  • Dataset: Synthetic + real images of road markings at varying wear levels, shot under controlled pulsed illumination with reference IRC device ground truth
  • Augmentation: Rain simulation, lens flare, motion blur, speed variation (20–120 kmph), night/fog conditions
  • Calibration: Transfer learning with 200+ reference readings from an NHAI-provided IRC handheld device
  • Target accuracy: Mean Absolute Error < 5 mcd/m²/lux against reference device
  • Continuous learning: Fleet data feeds back to improve model monthly

🎯 IRC Standard Integration

  • IRC 67-2012: Road signs — Class I (≥70), Class II (≥150), Class III (≥250) thresholds by sign type
  • IRC 35-2015: Pavement markings — white lines ≥100, yellow lines ≥150 (mcd/m²/lux)
  • System auto-detects marking type (sign vs lane vs stop line) via secondary classification head
  • Threshold lookup table is fully updateable without model retraining

☁️ Cloud Analytics Pipeline

  • Edge readings compressed and batched every 10 seconds, uploaded via 4G module
  • Server-side spatial clustering groups failures into maintenance zones (DBSCAN algorithm)
  • Priority scoring: failure density × road classification × last maintenance date
  • Auto-generates maintenance work orders ranked by urgency
  • Historical trend analysis predicts sign failure 30–60 days in advance
