System Architecture · Data Flow · IRC Standards Integration · Hardware Stack

🔧
Layer 1 — Hardware
💡 Pulsed LED Array
50W COB LED + Pulse Driver Circuit
Fires precisely timed white light bursts at the road surface. Pulsing at 50 Hz allows frame-differential ambient subtraction.
50W COB50 Hz PulseAmbient Subtraction
📷 Machine Vision Camera
Arducam 12MP IMX477 (Sony Sensor)
Fixed at IRC-standard 0.2° observation angle from illumination axis. Global shutter for motion-artifact-free capture at speed.
12MPIRC 0.2° AngleGlobal Shutter
🛰️ GPS + IMU Module
u-blox NEO-M8N (±2.5m accuracy)
Tags every frame with precise GPS coordinates and timestamp. IMU provides vehicle heading and tilt correction for measurement normalisation.
±2.5m10 Hz UpdateIMU Fusion
🌤️ Ambient Light Sensor
TSL2591 (0.1–88,000 lux)
Measures ambient illumination continuously. Feeds into AI model as a condition input, enabling the model to switch between day/night/fog processing branches.
0.1–88000 lux16-bitI²C
🏗️ Mounting Frame
Aluminium CNC Channel, Roof-Mounted
Mechanically fixes camera-LED geometry at the IRC observation angle. Rigid mount eliminates reading variance from vehicle vibration.
Fixed AngleVibration DampedAny Vehicle
🧠
Layer 2 — Edge AI
⚡ Edge AI Computer
NVIDIA Jetson Nano / Raspberry Pi 5
Runs the full inference pipeline at 15 fps. Handles LED pulse synchronisation, frame differencing, model inference, and GPS data fusion simultaneously.
15 fpsCUDA AcceleratedTFLite
🔬 Frame Differencing Engine
Software module — OpenCV + NumPy
Subtracts LED-off frame from LED-on frame per pixel, isolating only retroreflected light. Result is a clean luminance map of the road surface.
OpenCVPer-pixel DiffReal-time
🤖 CNN Regression Model
MobileNetV2 + Custom Regression Head
Takes the differenced frame + ambient lux + speed + condition flag as input. Outputs estimated RL value (mcd/m²/lux) + confidence interval. INT8 quantised for 4× speed.
MobileNetV2INT8 Quantised~18ms Inference
📋 IRC Standards Decision Engine
Python Rules Engine + Threshold DB
Compares predicted RL value against IRC 67 (signs) and IRC 35 (markings) thresholds. Classifies each reading as Pass / Warning / Fail. Triggers alerts on Fail events.
IRC 67IRC 353-class Output
🖥️ In-Vehicle Dashboard
7-inch Touchscreen + Web Dashboard
Displays live GPS heat map, current RL reading, status classification, and session statistics. Driver receives audio alert on Fail events.
Live Heat MapAudio AlertsTouchscreen
☁️
Layer 3 — Cloud
📡 4G/5G Data Uplink
Quectel EC21 LTE Module
Batches GPS-tagged readings every 10 seconds and uploads to cloud. Operates over existing mobile network — no special infrastructure needed on the highway.
4G/5G10s BatchCompressed
🗄️ Geospatial Data Store
PostGIS + TimescaleDB
Stores all readings with full GPS geometry and timestamp. Supports spatial queries — "show all Fail events within 500m of chainage 42+000" — for maintenance planning.
PostGISTime-seriesSpatial Query
🔍 Spatial Clustering Engine
DBSCAN Clustering Algorithm
Groups nearby failure readings into maintenance zones. Scores each zone by failure density × road classification × last maintenance date to prioritise work orders.
DBSCANPriority ScoringAuto Work Orders
📈 Predictive Failure Module
LSTM Time-series Model
Analyses historical RL trend per GPS zone to predict sign/marking failure 30–60 days in advance. Enables proactive maintenance instead of reactive repair after accidents.
LSTM30-60 Day ForecastProactive
🌐 NHAI Operations Dashboard
React + Mapbox GL JS Web App
Full-network heat map visible to all NHAI regional offices. Drill-down to individual readings, maintenance zones, trend charts, and auto-generated work orders for field teams.
ReactMapboxMulti-regionITMS Integration

Calibration Note: The AI regression model outputs relative luminance that is calibrated against an IRC-compliant reference handheld retroreflectometer during initial setup. A calibration run of 200+ reference readings establishes the transfer function from image luminance to mcd/m²/lux. NHAI can provide their existing handheld device for this one-time calibration, after which the vehicle-mounted system operates autonomously. The calibration is re-checked every 6 months or after any hardware change.
